## 一句话结论

当前站点的核心问题不是“没有内容”，而是“内容很多，但用户下一步经常断掉”。从第一性原理看，家长访问这类页面时要解决四个问题：我是否被理解、这个项目是否可信、我现在应该做什么、做完之后会发生什么。现在前两项较强，后两项明显不足。

## 用户旅程地图

### 1. 新家长从首页进入

用户动机：孩子有手机、作业、情绪、动力、青春期沟通等问题，希望先判断这里是否理解自己。

实际路径：

- 首页首屏提供三个入口：`/life`、`/summer-hua`、`/#echo`。
- “也许，你是从一个具体处境来到这里”下的六张处境卡都使用 `href="#"`，点击不会进入对应解决路径。
- 主导航“生态回声”指向 `/#voices`，但首页实际锚点是 `#echo`。
- 主导航“当前开放”指向 `/summer-hua/apply`，该页面返回 404。
- 底部“免费咨询”指向 `/contact`，该页面有表单。

判断：情绪共鸣强，但路径转化弱。用户看到自己后，没有被带到“适合我的下一步”。

### 2. 对暑假华学苑感兴趣的家长

用户动机：想知道 28 天值不值得、适不适合孩子、安全吗、怎么报名。

实际路径：

- `/summer-hua/` 是完整长页，包含理念、课程、四阶段、学部、真实回声、老师团队、FAQ、适合/不适合、流程。
- 页面内“预约老师一对一咨询”会打开弹窗，提供微信 `tianshizaixian68` 和电话 `15829900194`。
- 页面说明报名步骤：先沟通、提交问卷、一对一面谈、确认报名。
- 但主站导航的“当前开放”仍然指向 404 的 `/summer-hua/apply`。
- 费用、退款、医疗安全、突发情况、家长授权、隐私边界等高决策信息不够集中。

判断：这是当前最完整的转化页，但关键报名入口没有统一到一个稳定 URL，用户从全站导航进入会断。

### 3. 读《生命成长力》或对理念感兴趣的用户

用户动机：先理解理念，看看是否能在家庭里试一小步，再决定是否买书、共读、咨询或进入华学苑。

实际路径：

- `/life` 有真实回声、作者回应、处境卡、小步练习、共读、章节导图、实践线说明。
- 子站又叠加了全站头部和独立头部；底部也出现子站 footer + 全站 footer。
- 子站导航中 `#problems` 没有对应 section，实际相近 section 是 `#situations`。
- “购买书籍”使用 `href="#小程序://群团团接龙pro/MKpxHaHozzLx4Rd"`，在 Web 中不是可用购买链接，还可能被页面的平滑滚动逻辑错误拦截。
- 页面没有清晰的“加入共读 / 咨询 / 购买 / 下载练习卡”的稳定落点。

判断：内容很适合建立信任，但最终行动路径缺失。读者会被照见，但不一定知道下一步去哪。

### 4. 已报名或已有服务关系的家庭

用户动机：进入家长服务、查看孩子档案或活动记录。

实际路径：

- 全站头部“家庭入口”指向 `/portal`。
- `/portal` SSR 内容只有加载状态，标题是“天使在线 - 家长服务”，描述是“查看孩子成长档案与活动记录”。
- 对未登录或新用户来说，没有解释这是已报名家庭入口、需要登录、如何获得账号、忘记入口怎么办。

判断：入口名称对老用户可能有用，对新用户会制造分心；它出现在主导航最强 CTA 位置，会抢走咨询/报名动作。

## 第一性原理分析

### 用户不是来“阅读网站”，而是来降低决策风险

对家长来说，教育项目尤其是 28 天住宿制项目，是高风险决策。页面必须尽快回答：

- 我的孩子适不适合？
- 安全和情绪风险怎么处理？
- 老师是谁，是否可信？
- 费用、退款、报名资格是什么？
- 如果不适合，有没有替代路径？
- 我现在点哪里，点完后谁会联系我？

目前页面大量回答了“理念是什么”和“为什么重要”，但对“我下一步怎么走”回答不稳定。

### 信任来自一致性，不只来自故事

站点有大量真实故事、老师介绍和项目细节，这是优势。但信任也会被小断点快速消耗：404、空链接、锚点不跳、数据口径不一致、购买链接不可用、旧版 About 的 stock 风格。这些问题会让用户怀疑“项目很认真，但网站还没准备好”。

### 转化不是一个按钮，而是一条连续路径

当前有三套转化逻辑：

- `/summer-hua/`：微信/电话弹窗。
- `/contact`：咨询表单。
- `/life`：小程序购买链接，但 Web 不可用。

建议收敛为一个主路径：`了解 -> 判断适合 -> 预约咨询/提交问卷 -> 面谈 -> 确认报名`。不同页面只是在不同阶段把用户送入同一条路径。

## 优先级建议

### P0：当天修复

1. 修复 `/summer-hua/apply` 404。短期可 301 到 `/summer-hua/#trio` 或 `/contact?source=summer-hua`；长期建立正式 `/apply`。
2. 修复所有空链接和错误锚点：`/#voices`、`#problems`、首页六张处境卡、底部“家庭入口”。
3. 修复 `/life` 买书链接：不要用 `#小程序://...`；提供真实 URL、二维码图片、微信复制、备用咨询。
4. 全站主 CTA 改成“预约咨询”或“判断是否适合”，不要把“家庭入口”放在最强按钮位置。

### P1：一周内修复

1. 建立统一转化页 `/consult` 或 `/apply`：
   - 孩子年级/年龄
   - 当前困扰
   - 意向项目
   - 电话/微信
   - 隐私说明
   - 提交后 24 小时内联系
   - 微信/电话兜底
2. 首页六个处境卡做成分流入口：
   - 作业/拖延 -> `/life#situations`
   - 手机/屏幕 -> `/life#practice`
   - 没劲/空洞 -> `/summer-hua#fit`
   - 青春期沟通 -> `/life#echoes`
   - 活动后承接 -> `/summer-hua#after`
   - 父母越努力越紧 -> `/life#author-response`
3. `/summer-hua/` 首屏后增加“关键信息卡”：
   - 对象：三年级至高中
   - 时间：2026-07-12 至 2026-08-08
   - 地点：西安秦岭/终南山脚下
   - 形式：28 天住宿制
   - 下一步：先咨询、再问卷、再面谈
4. `/life` 增加三个稳定 CTA：
   - 购买书籍
   - 加入共读
   - 和老师聊聊我家的情况
5. `/portal` 改为“已报名家庭入口”，未登录页说明使用场景。
6. 修复 `/camps` 活动详情 404，或先从导航弱化该入口。

### P2：后续迭代

1. 统一品牌叙事和数据口径：服务家庭、服务学员、课程活动、创立时间。
2. 压缩长页信息层级：先给决策摘要，再给故事和理念深读。
3. 增加费用、退款、医疗、心理危机、不适合情况、家长联系机制、隐私授权说明。
4. 给微信/电话增加复制按钮；二维码不要依赖第三方临时生成服务。
5. 加埋点：首屏 CTA 点击、锚点点击、弹窗打开、电话点击、微信复制、表单提交、表单失败、404 来源。

## 证据记录

- `/` 返回 200，HTML 约 90 KB。
- `/summer-hua/` 返回 200，HTML 约 168 KB。
- `/life` 返回 200，HTML 约 63 KB。
- `/summer-hua/apply` 返回 nginx 404。
- `/contact` 返回 200；前端表单提交到 `/api/inquiries/submit`，成功状态期望为 201。
- `/portal` 返回 200，但 SSR 首屏为加载状态，适合已登录应用入口，不适合作为新用户主 CTA。
- `/camps` 返回 200；活动 API 返回 1 个活动，但详情 `/camps/act-demo-001` 返回 404。
- `/api/activities/categories` 返回 401，会影响活动分类数据。
- `/api/content/brand/home_support` 返回 `data:null`，首页“近期支持”依赖 fallback。
- `/articles` 301 到 `/articles/index.html`，下载约 1.68 MB HTML。

## 当前不建议做的事

- 不建议继续堆更多故事，先修路径闭环。
- 不建议把所有 CTA 都导向微信，至少保留一个 Web 表单和一个稳定 URL。
- 不建议让新用户先进入 `/portal`，这更像服务后台，不是销售/咨询入口。
- 不建议在 `/life` 里只放理念，不放可执行下一步；读者需要从“被照见”过渡到“我现在可以做什么”。
