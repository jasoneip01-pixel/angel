## 一、产品公理拆解

### 1.1 这是什么？

**信任构建机**——不是信息展示站，不是营销落地页。

家长的核心动作不是"阅读"，是做一个高风险决策：把11岁的孩子交给陌生人28天。网站的唯一存在理由是把这个决策的不确定性降到可接受阈值。

### 1.2 用户四层需求

| 层 | 需求 | 当前评分 | 说明 |
|:--:|------|:--:|------|
| L1 情绪 | 我被理解了吗？ | 🟢 9/10 | 文案姿态是核心竞争力 |
| L2 信任 | 这个机构可信吗？ | 🟡 7/10 | 故事强，面孔弱，抽象数字多 |
| L3 决策 | 适合我的孩子吗？ | 🟢 8/10 | 坦诚说"不适合"，值得加分 |
| L4 行动 | 我现在点哪里？点完发生什么？ | 🟡 6/10 | 路径存在，但有断点和摩擦力 |

### 1.3 核心悖论

最有转化力的东西不是文案、不是设计，是**姿态**：不推销、不制造焦虑、先说"你可能很累"、"试一次、试一半、明天再试，都可以"。这个姿态对高焦虑家长是解毒剂。但同时——这个姿态也需要明确的下一步，否则用户从"被照见"到"我知道点哪里"之间会迷路。

## 三、内容/文案评估

### 3.1 做得最好的三句话

1. **"你不需要在这里找标准答案。你只需要一件事：先把那个急着找答案的自己，停一下。"** — 这句话在第一性原理上完全正确。家长来的时候是"我必须解决问题"的模式，而产品本质是"先被看到，再找路"。这句文案把产品价值映射到了用户情绪状态上。

2. **"变化不是被说服出来的，是在时间里慢慢露出来的。"** — 这句话回答了"为什么是28天"这一核心质疑，同时定义了产品的底层信念。

3. **"不是好评墙，是家庭亲历的变化。有进步，也有反复。正因为不完美，才可信。"** — 反向操作，用不完美建立信任，非常精准。

### 3.2 需要改进的文案

- "20万+服务家庭" → 改成"从2015年开始，陪伴了XX个家庭走过XX年"（时间+过程 > 绝对数字）
- Hero标题"在山水与经典中，遇见更好的自己" → 太泛。改成具体的，如"28天，让孩子重新发现自己可以做到的事"或"真实生活里，孩子正在发生的那些变化"
- "预约咨询"/"判断是否适合"在多个位置出现但措辞不完全一致 → 统一主CTA文案

### 3.3 信息缺口

| 缺失项 | 重要性 | 说明 |
|------|:--:|------|
| 费用区间 | 🔴 高 | 全文无任何价格信息，唯一提到"费用"在FAQ里说"按当期规则确认"，用户必须填表才能知道——增加流失 |
| 营地实景照片 | 🔴 高 | 只在image-grid有三张小图，没有室内、宿舍、食堂、教室的实景 |
| 老师面孔 | 🔴 高 | teachers区域只有姓名+一句话，没有照片 |
| 安全/医疗细节 | 🟡 中 | FAQ有提及但极度简略 |
| 退费政策 | 🟡 中 | 只说"按当期规则"，无任何透明度 |

## 四、UI/设计评估

### 4.1 设计系统

**配色**：暖纸白(#fffdf8) + 奶油(#f6efe3) + 暗绿(#284c3d) + 暗红(#b74332) + 金色(#c5933d) — 北欧有机现代主义路线，克制、暖、不甜腻。✅

**字体**：系统字体栈，避免额外下载。✅

**间距/节奏**：section 96px padding, card 24px padding, grid 20px gap — 比例舒适。✅

**阴影**：`--shadow: 0 18px 50px rgba(54,39,25,0.1)` — 暖调阴影，不是冷灰。细节用心。✅

### 4.2 问题

**1. 字重爆炸（P1）**

CSS中出现了至少7种字重值：`920, 880, 850, 820, 780, 760, 720`。font-weight 850/880/920在大多数系统字体中不存在，会fallback到700或900。视觉上实际只有2-3个层级。建议显式收敛为三层：
```css
--fw-heading: 900;   /* h1 */
--fw-strong: 700;    /* h2, h3, strong */
--fw-body: 400;      /* 正文 */
```

**2. Hero遮罩可读性（P1）**

`hero-immersive::before`叠加了90deg+180deg+底部三层渐变遮罩。在亮色图片上效果OK，但如果换了图片（或用户在低对比度屏幕上），文字对比度可能不足。建议给h1增加更强的text-shadow。

**3. 卡片缺乏信息层级（P2）**

所有.card共享同一视觉权重：白底+细边框+相同阴影。用户无法快速区分"这是内容卡"和"这是行动卡"。建议给转化类卡片（CTA卡片）一个微妙的渐变或左边框强调。

**4. 图片严重不足（P0）**

`assets/images/`目录只有8张图。`article-001.jpg`被三个页面复用为hero背景。没有一张人脸照片。对于教育+住宿类产品，这是信任构建的最大短板。

**5. sticky-cta缺少上下文感知（P2）**

移动端底部固定三按钮（电话/微信/咨询），在所有页面、所有滚动位置都存在。用户在深度阅读FAQ或章节导图时，这68px是纯粹的信息浪费。建议：向下滚时隐藏，向上滚时出现。

## 五、用户旅程评估

### 5.1 四个关键路径

**路径A：焦虑家长 → 判断适合 → 预约咨询**

当前：首页Hero → 三个CTA并列 → 处境卡分流 → consult表单  
摩擦点：首屏CTA过多（3个），处境卡链接到子站锚点（需要子页加载后才能定位），表单提交后等待"24小时内联系"——相比即时微信/Tel，这个等待是流失点。

**路径B：感兴趣家长 → 了解28天 → 决定报名**

当前：summer-hua页 → 全页浏览 → 底部"联系老师"  
这是最完整的转化页。流程清晰，FAQ到位。唯一的问题：从"被说服"到"行动"之间缺少一个低摩擦的中间步骤（如"先做5题自测"）。

**路径C：对理念感兴趣 → 买书 → 进入共读**

当前：life页 → 浏览回声 → 买书需要填表获取链接  
核心问题：买书是低摩擦动作（几十块钱），不应该和"预约咨询"（需要准备孩子信息、等老师联系）共用同一流程。应该提供直接购买入口（小程序码/二维码）。

**路径D：已报名家庭 → 查看档案**

当前：portal页 → 说明 + "需要账号协助"  
作为已报名入口，信息足够。但放在主导航的"已报名家庭入口"对未报名用户是噪音。

### 5.2 断点评级

| 断点 | 严重度 | 现状 |
|------|:--:|------|
| 买书路径强制走咨询 | 🔴 P0 | life页"购买书籍"指向consult而不是直接购买 |
| 费用信息完全缺失 | 🟡 P1 | 全文无任何价格信息 |
| 无即时价值交付 | 🟡 P1 | 用户在留资前得不到任何可带走的东西 |
| 人脸/实景照片缺失 | 🔴 P0 | 信任构建的物理证据缺失 |
| 首屏三CTA并列 | 🟡 P2 | 增加用户认知负担 |
| sticky-cta无滚动感知 | 🟢 P3 | 体验问题，不影响转化 |

## 八、优先级建议

### P0 — 立即

1. **增加实景照片**：营地室内/宿舍/食堂/教室至少各一张，老师团队每人一张正面照
2. **买书路径独立化**：提供小程序码或二维码直接购买，不与咨询混用同一流程
3. **Hero图片替换**：首页用营地实景替代article-001.jpg

### P1 — 本周

4. **费用透明度**：至少在FAQ或summer-hua关键信息区给出费用区间或"咨询获取"的明确说明
5. **即时价值交付**："3分钟判断孩子适不适合28天"的5题自测（即时结果，不需留资）
6. **字重收敛**：7种→3种
7. **首屏CTA收敛**：主CTA一个（"先说说孩子情况"），次要链接一个（"先看别人家的真实变化"）

### P2 — 后续迭代

8. sticky-cta加滚动感知
9. 卡片信息层级视觉区分
10. OG标签 + 结构化数据
11. 图片格式优化（webp）

## 九、方法论声明

本审计基于以下公理：
- **不依赖Codex的任何输出或分支**：仅读取源码和线上站点
- **不假设"Codex会处理"**：每个发现独立评估，不因"Codex在重建"而跳过

- **反方论点已纳入**：本报告最可能被反驳的点——"费用不透明是故意的（先建立信任再说价格），买书走咨询是引导用户进入更高价值路径"——这两个反方有合理性，但不足以抵消透明度缺失和路径摩擦带来的流失
